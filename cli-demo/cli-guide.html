Welcome to unhazzle CLI - Ecommerce App Deployment Guide

This guided tutorial will walk you through deploying a complete ecommerce application with database and cache services.

Let's get started with: unhazzle help

This will show you all available commands to get started.
Step 1: Login with GitHub

Authenticate with your GitHub account:

unhazzle login This connects your GitHub repositories for deployment.
Step 2: Initialize Project

Create a new Unhazzle project:

unhazzle init --name my-project --env dev --app-name my-app --with-db --with-cache --app-image ghcr.io/my-org/my-app:v2.1.4 --health-check /health --min-instances 1 --max-instances 3 Available flags with defaults:
--name (my-project), --env (dev), --app-name (my-app), --app-image (ghcr.io/my-org/my-app:v2.1.4), --app-cpu (0.5), --app-memory (512Mi), --public (true), --with-db (false), --db-engine (postgres), --db-size (small), --with-cache (false), --cache-engine (redis), --cache-size (small), --health-check (/health), --min-instances (1, app only), --max-instances (3, app only), --with-github-actions (false).
Step 3: Deploy Infrastructure

Apply all infrastructure changes:

unhazzle apply This deploys everything to production with cost estimation.
Step 4: Check Status

Verify your deployment:

unhazzle status This shows the health and status of all your services.
Step 5: Monitor Application Logs

Start monitoring application logs:

unhazzle logs This starts streaming application logs in real-time. Run the command again to stop log streaming.
Step 6: Check Application Metrics

Monitor your application's performance metrics:

unhazzle metrics This displays real-time metrics for your application, database, and cache services.
Step 7: Destroy Infrastructure

Destroy all infrastructure resources:

unhazzle destroy This will prompt for confirmation by requiring you to type the project name. Use with extreme caution!
