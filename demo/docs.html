📚 unhazzle Documentation

Welcome to unhazzle! Deploy and manage your cloud infrastructure with a visual-first approach.

💡 New to unhazzle? Start with our Quick Start Guide to deploy your first application in under 5 minutes.

Quick Start

Get started with unhazzle in three simple steps:

1. Connect Your Repository

Link your GitHub, GitLab, or Bitbucket repository. unhazzle automatically detects your application type and suggests optimal configurations. 

# Example repository URL
https://github.com/your-username/my-awesome-app

2. Configure Resources

Use the visual canvas to drag and drop resources, or edit the YAML manifest directly. Both approaches are synced in real-time. 

version: '1.0'
project: my-project
environment: dev

resources:
  - name: my-app
    type: application
    source:
      repo: github.com/your-username/my-awesome-app
      dockerfile: ./Dockerfile
    resources:
      cpu: 1
      memory: 512M
    env:
      - name: DATABASE_URL
        valueFrom:
          resource: my-db
          key: connectionString

  - name: my-db
    type: database
    engine: postgres
    version: '16'
    size: 1G
    backups: true
    retention: 7d

3. Deploy

Review your cost estimation and click "Apply Changes". Your infrastructure will be provisioned in under 2 minutes.

✅ Deployment Complete! Your application is now live at my-app.unhazzle.dev

Authentication

unhazzle uses OAuth 2.0 for secure authentication. Currently supported providers:

  • GitHub - Sign in with your GitHub account
  • GitLab - Use your GitLab credentials
  • Google - Sign in with Google (coming soon)

Applications

Deploy containerized applications from any Git repository. unhazzle supports:

  • Automatic builds from Dockerfile
  • Buildpack detection for popular frameworks (Node.js, Python, Ruby, Go, etc.)
  • Zero-downtime deployments with automatic rollback
  • Horizontal scaling with load balancing
  • Custom domains with automatic SSL certificates

Configuration Options

Applications support the following configuration parameters: 

- name: my-app
  type: application
  source:
    repo: github.com/user/repo
    branch: main              # Optional, defaults to main
    dockerfile: ./Dockerfile  # Optional, auto-detected
  resources:
    cpu: 1                    # vCPU cores (0.5 - 8)
    memory: 512M              # RAM allocation
    instances: 1              # Number of replicas (1 - 10)
  ports:
    - port: 8080              # Internal port
      public: true            # Expose to internet
  env:
    - name: NODE_ENV
      value: production
  healthCheck:
    path: /health             # Health check endpoint
    interval: 30s             # Check frequency

Databases

Provision fully managed databases with automatic backups, point-in-time recovery, and high availability.

Supported Engines

  • PostgreSQL - Versions 12, 13, 14, 15, 16
  • MySQL - Versions 5.7, 8.0, 8.4
  • MariaDB - Versions 10.6, 10.11, 11.4
  • MongoDB - Versions 5.0, 6.0, 7.0
⚠️ Important: Database instances cannot be downgraded after creation. Choose your storage size carefully or contact support for migrations.

Backup Strategy

All databases include automatic backups:

  • Frequency: Daily automated backups
  • Retention: Configurable from 3 to 30 days
  • Point-in-Time Recovery: Restore to any second within retention period
  • On-Demand Backups: Create manual backups anytime

Caches

Deploy high-performance in-memory caches to accelerate your applications.

Supported Engines

  • Redis OSS - Open-source Redis (latest stable)
  • Valkey - Redis fork with enhanced features
  • Memcached - Distributed memory caching

Use Cases

  • Session storage
  • API response caching
  • Real-time leaderboards
  • Pub/sub messaging
  • Rate limiting

YAML Manifest

The unhazzle.yaml file is your infrastructure as code. Commit it to your repository for GitOps workflows.

File Structure

version: '1.0'           # Manifest version
project: project-name    # Unique project identifier
environment: dev         # Environment name (dev, staging, prod)

resources:               # List of infrastructure resources
  - name: resource-1
    type: application
    # ... configuration

  - name: resource-2
    type: database
    # ... configuration
💡 Pro Tip: Use environment variables in your YAML with ${ENV_VAR} syntax for dynamic configuration across environments.

CLI Installation

Install the unhazzle CLI to manage your infrastructure from the terminal.

macOS / Linux

curl -sSL https://get.unhazzle.io | bash

Windows

iwr https://get.unhazzle.io/install.ps1 -useb | iex

Verify Installation

unhazzle --version
# Output: unhazzle CLI v1.2.3

CLI Commands

Essential commands for managing your infrastructure:

Authentication

# Log in to your account
unhazzle login

# Log out
unhazzle logout

# Check authentication status
unhazzle whoami

Project Management

# List all projects
unhazzle projects list

# Create a new project
unhazzle projects create my-project

# View project details
unhazzle projects info my-project

Deployment

# Apply changes from unhazzle.yaml
unhazzle apply

# Apply with automatic approval
unhazzle apply --auto-approve

# View deployment plan without applying
unhazzle plan

# Rollback to previous version
unhazzle rollback

Resource Management

# List all resources
unhazzle resources list

# Get resource details
unhazzle resources get my-app

# View logs
unhazzle logs my-app --follow

# Scale application
unhazzle scale my-app --instances 3

Scaling

unhazzle supports both vertical and horizontal scaling for your applications.

Horizontal Scaling

Increase the number of application instances to handle more traffic: 

resources:
  - name: my-app
    type: application
    resources:
      instances: 3        # Run 3 replicas
      cpu: 1
      memory: 512M
    autoscaling:
      enabled: true
      minInstances: 2
      maxInstances: 10
      targetCPU: 70       # Scale when CPU > 70%

Vertical Scaling

Increase CPU and memory resources for individual instances: 

resources:
  - name: my-app
    type: application
    resources:
      cpu: 4              # Increase to 4 vCPUs
      memory: 8G          # Increase to 8GB RAM

Monitoring & Observability

Built-in monitoring and alerting for all your resources:

Metrics Available

  • Applications: Request rate, response time, error rate, CPU, memory
  • Databases: Connection count, query performance, storage usage
  • Caches: Hit rate, memory usage, evictions, command latency

Alerting

Configure alerts for critical events: 

alerts:
  - name: high-error-rate
    condition: error_rate > 5%
    duration: 5m
    channels:
      - email
      - slack

CI/CD Integration

Integrate unhazzle with your CI/CD pipeline for automated deployments.

GitHub Actions

name: Deploy to unhazzle

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Deploy to unhazzle
        uses: unhazzle/deploy-action@v1
        with:
          api-token: ${{ secrets.UNHAZZLE_TOKEN }}
          project: my-project
          environment: production
